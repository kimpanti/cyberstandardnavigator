How Mappings Are Derived: Mappings in this engine utilize Ontological Alignment. Since frameworks like ITIL (Processes) and NIST 800-53 (Technical Controls) serve different purposes, we map them based on Security Intent.
  • Direct Mapping: High semantic overlap (e.g., NIST AC-2 vs ISO A.5.15).
  • Functional Equivalent: The process required to satisfy a technical requirement (e.g., ITIL Change Enablement facilitates NIST CM-3).
